# 智投 AI — 全方位 AI 投資分析系統

一套整合即時市場資料、總體經濟數據與 AI 大型語言模型的投資輔助工具。透過網頁介面，讓使用者不需要寫程式，就能完成個股分析、估值計算、技術指標解讀、黑天鵝風險偵測、總經環境判斷、投資組合健診、回測模擬與月度市場摘要等工作。

---

## 系統需求

- Node.js 18 版以上
- 至少一個 AI API Key：Anthropic（Claude）或 OpenAI（GPT-4o）

---

## 本地安裝與啟動

第一步，安裝相依套件：

```
npm install
```

第二步，在專案根目錄建立 `.env` 檔案：

```
ANTHROPIC_API_KEY=你的 Claude API Key
OPENAI_API_KEY=你的 OpenAI API Key
FRED_API_KEY=你的 FRED API Key
```

三個 Key 中，AI Key 至少填一個，FRED_API_KEY 選填（無法取得 Fed 利率、CPI 等數據，但 VIX 仍可顯示）。

第三步，啟動伺服器：

```
npm start
```

開啟瀏覽器前往 `http://localhost:3000`。

## 部署到 Render

Render 是免費的雲端部署平台，部署後會有公開網址。

1. 將專案上傳到 GitHub Repository
2. 前往 render.com 註冊並登入
3. 點擊 New，選擇 Web Service
4. 選擇你的 GitHub Repository，點擊 Connect
5. 填入設定：
   - Build Command：`npm install`
   - Start Command：`npm start`
   - Instance Type：Free
6. 新增環境變數：

   | 變數名稱 | 說明 | 是否必填 |
   |---|---|---|
   | ANTHROPIC_API_KEY | Claude API Key | 二擇一必填 |
   | OPENAI_API_KEY | OpenAI API Key | 二擇一必填 |
   | FRED_API_KEY | 美聯儲經濟數據 API Key | 選填 |

7. 點擊 Create Web Service，約 2 至 3 分鐘完成部署

注意：Render 免費方案在閒置 15 分鐘後會休眠，下次開啟需等約 30 秒喚醒。如果分析功能按下去沒有反應，通常是伺服器正在喚醒，稍等後再試即可。

## 功能模組說明

### 一、AI 對話理專

主要對話介面，用中文詢問任何投資問題，AI 以串流方式即時回覆。同一個 session 具有對話記憶，可以連續追問。

左側欄提供六個快速詢問按鈕：今日大盤分析、熱門投資題材、投資組合規劃、基本面指標、技術分析入門、存股策略。

自選股區域顯示五檔股票的即時報價（台積電、聯發科、鴻海、緯創、中華電），點擊直接發問。

右側面板提供快速分析工具，支援個股分析、新聞解讀、組合評估、大盤展望四種模式。

### 二、模組化 AI 提問範本庫

收錄 16 個經過設計的專業提問範本，分為四類：

**財報分析**
季報深度分析（營收、毛利率、EPS、自由現金流）、年報財務健康度（負債比、ROE 趨勢、應收帳款週轉率）、EPS 成長分析（近五年趨勢與未來預估）、殖利率存股評估（股利穩定性與配發能力）。

**選股操作**
成長股篩選條件（PEG ratio、護城河評估）、AI 自動估值分析（多方法估值）、產業體質篩選（供需分析、競爭地位）、投資組合配置 Prompt（依風險偏好設計比例）。

**精準進退場**
股價趨勢分析系統（均線、趨勢通道、支撐壓力位）、技術指標買入訊號（KD、RSI 綜合判斷）、停損停利策略設定（技術面與固定比例停損）、回測策略評估。

**整合分析**
個股完整健診報告（基本面、技術面、籌碼面、估值一次完成）、產業輪動分析（資金流向與下一波輪動方向）、總經環境投資評估（利率、通膨對股市的影響）、投資組合整體健診。

每個範本可點擊「套用到對話」帶入對話框，或點擊「複製」貼到其他工具使用。

### 四、AI 自動化估值模組（含 DCF 與 PE Band）

輸入股票代號後，系統自動從 Yahoo Finance 取得即時股價、EPS、自由現金流，並從五年歷史數據計算 PE Band，再由 AI 執行完整估值。

**DCF 現金流折現模型**

系統自動讀取公司近期自由現金流（Free Cash Flow），根據用戶輸入的盈餘成長率，預測未來五年現金流，並以 WACC 10% 折現計算現值，加上終值（終值成長率 2.5%），得出公司總估算價值與每股內在價值。

**PE Band 本益比區間分析**

從五年月線歷史股價自動計算：歷史最低本益比、最高本益比、目前本益比在五年區間的百分位位置。幫助判斷目前股價是偏貴還是偏便宜。

**AI 整合分析內容**

多種估值法比較（PE 法、PB 法、殖利率法、DCF 法、PEG 法）、安全邊際評估、建議買入區間（強力買入價、合理買入價、觀察等待價）、買入或賣出建議。

---

### 五、技術指標分析模組

輸入技術指標數值，系統在呼叫 AI 前先自動從 Yahoo Finance 抓取近 30 日每日收盤價與成交量，一併提供給 AI 讓分析更準確。

需要輸入的欄位：股票代號、目前股價、MA5、MA20、MA60、成交量趨勢、KD 的 K 值與 D 值、RSI、近期走勢描述。

AI 分析內容：均線多空排列（多頭、空頭、整理格局）、KD 指標解讀（超買超賣、黃金交叉死亡交叉）、RSI 強弱評估、重要支撐位與壓力位、短中長期趨勢、進場點、停損點、目標價。

---

### 六、回測分析模組（含 Sharpe Ratio 與 Max Drawdown）

描述交易策略，AI 模擬回測績效。系統同時從 Yahoo Finance 取得真實歷史數據，計算該標的在回測期間的實際量化指標作為參考基準。

**真實計算的量化指標**

系統從 Yahoo Finance 抓取回測期間的每日收盤價，計算以下指標並顯示在結果上方：

夏普比率（Sharpe Ratio）：超額報酬除以波動率，衡量每承擔一單位風險所獲得的報酬。小於 0 代表不如無風險利率，0 至 1 為尚可，1 至 2 為良好，大於 2 為優秀。

最大回撤（Max Drawdown）：持有期間從高點到低點的最大跌幅。代表「最壞情況下你會虧損多少」。

卡瑪比率（Calmar Ratio）：年化報酬除以最大回撤，衡量每承擔一單位最大虧損所獲得的年化報酬。

年化波動率：每日報酬率的標準差乘以 252 的平方根，代表市場波動程度。

年化報酬率：以被動買進持有為基準的實際報酬率。

上漲日勝率：收盤上漲的交易日佔比。

**AI 分析內容**

策略邏輯合理性評估、預估年化報酬率與夏普比率、回測失敗案例分析（列出三種市場環境下策略失效的情境與原因）、與大盤被動持有比較、策略優化建議、適合與不適合的市場環境。

---

### 七、投資組合配置

輸入持股明細，AI 分析組合健康程度並提供最佳化建議。

設定欄位：風險承受度（保守型、穩健型、積極型、高積極型）、投資目標（穩健成長、高殖利率收息、長期資本增值、退休規劃）、投資期間。

AI 分析內容：集中度與多元性評分、產業與地區配置分析、建議配置比例、減碼與增碼說明、推薦新標的、預期風險收益改善幅度。

### 八、自動化投資報告

選擇報告期間（本週至全年）與重點方向，輸入持股明細，生成包含完整章節的投資分析報告，可複製後貼入 Word 或 Notion 存檔。

報告章節：執行摘要、市場環境回顧、持股績效分析、風險指標評估、重要事件影響、下期展望與策略、調整建議、免責聲明。

### 九、每月重點摘要推播

選擇月份，AI 生成該月台股市場的完整摘要，包含大盤表現、前三強勢與前三弱勢產業、重要財經事件、外資投信自營商籌碼動向、本月最強個股、下月關鍵觀察指標、風險警示。

點擊「繼續在 AI 討論」可將摘要帶入對話模組，請 AI 提供下個月的投資策略建議。

### 十、宏觀經濟數據關聯模組

整合多個總體經濟指標的即時數據（需 FRED API Key），讓 AI 判斷目前處於哪個經濟週期，並分析對台股的影響。

**顯示的總經指標**

聯邦基金利率（FEDFUNDS）：聯準會的基準利率，直接影響全球資金成本。
CPI 消費者物價指數（CPIAUCSL）：衡量通膨水準，聯準會政策決策的重要依據。
PCE 個人消費支出物價指數（PCEPI）：聯準會最偏好的通膨衡量指標。
美國失業率（UNRATE）：反映勞動市場熱度，影響聯準會升降息時機。
10 年期美國國債殖利率（DGS10）：影響股票估值的無風險利率基準。
非農就業人數（NFP）：美國每月新增就業人數，判斷經濟擴張或收縮。
VIX 恐慌指數：市場情緒的即時溫度計。

**AI 分析內容**

經濟週期判斷（擴張期、過熱期、衰退期、復甦期）、聯準會政策展望（未來 6 至 12 個月升息或降息概率）、通膨與就業趨勢分析、市場風險評估、對台股科技類股與出口導向企業的具體影響、建議資產配置方向、未來三個月需關注的重大總經事件。

注意：若未設定 FRED_API_KEY，頁面會顯示友善提示，說明如何免費申請，VIX 數據仍可正常顯示。

### 十一、黑天鵝預警系統

基於滾動 Z-Score 統計模型，偵測 VIX 與個股的異常波動，自動計算風險等級。

**演算法說明**

系統計算過去 252 個交易日（約一年）的滾動 20 日均值與標準差。當某一天的數值偏離 20 日均值超過 2.5 個標準差（Z 值大於等於 ±2.5）時，視為統計異常。

Z 值的意義：
- Z 值介於 -2.5 至 +2.5：正常範圍
- Z 值大於 ±2.5：異常，觸發警報
- Z 值大於 ±3.0：極端異常（三個標準差事件），需高度關注

系統同時偵測兩個維度：日報酬率異常（價格異常波動）和成交量異常（異常大量或異常低量）。兩者同時觸發時，黑天鵝風險大幅提升。

**使用方式**

輸入最多三個台股代號，點擊執行異常偵測。系統需約 10 至 15 秒從 Yahoo Finance 抓取一年歷史數據並計算。

**結果顯示**

每個標的顯示目前 Z 值、成交量 Z 值、風險等級（LOW、MEDIUM、HIGH）、近期異常事件列表（日期、異常幅度、方向）。

**AI 分析內容**

整體風險評級、VIX 當前水準分析、個股異常解讀、歷史類比（此 Z 值水準的歷史後續發展）、具體預警建議（是否降低倉位、增加避險部位）、黑天鵝情境的風險緩解策略。

## 常見問題

**按下分析按鈕後沒有反應，或顯示「伺服器錯誤」**

最常見原因是 Render 環境變數未正確填入 API Key。請至 Render Dashboard 的 Environment 頁面，確認 ANTHROPIC_API_KEY 或 OPENAI_API_KEY 已填入且沒有多餘的空格。

**使用 Render 免費方案時，第一次按鈕沒反應**

Render 免費方案在閒置 15 分鐘後會休眠，再次開啟需要約 30 秒喚醒。等待後再試即可。若不想有休眠問題，可升級至付費方案（約每月 7 美元）。

**宏觀經濟模組顯示「FRED API 未設定」**

前往 fred.stlouisfed.org 免費申請 FRED API Key，然後在 Render Dashboard 的 Environment 新增 FRED_API_KEY 變數。申請完全免費，約 5 分鐘完成。

**黑天鵝預警需要 10 至 15 秒**

這是正常現象。系統需要從 Yahoo Finance 抓取三個標的各一年的每日歷史數據（約 750 至 756 筆），再進行 Z-Score 計算，等待時間屬正常範圍。

## 專案檔案結構

```
ai-investment/
├── server.js          後端伺服器，處理所有 API 請求與數據計算
├── package.json       Node.js 相依套件設定
├── render.yaml        Render 部署設定
├── .gitignore         Git 忽略清單（包含 .env 與 node_modules）
├── README.md          本說明文件
└── public/
    └── index.html     前端介面（單頁應用，包含全部 UI 與 JavaScript）
```

**server.js 主要功能**
- 與 Yahoo Finance 取得即時股價、歷史走勢、財務數據
- 向 TWSE 證交所 API 取得外資買賣超
- 向 FRED API 取得總體經濟指標
- 向 BLS 取得非農就業數據
- 執行 Z-Score 異常偵測演算法（黑天鵝模組）
- 執行 DCF 現金流折現計算（估值模組）
- 執行 PE Band 歷史區間計算（估值模組）
- 計算 Sharpe Ratio、Max Drawdown、Calmar Ratio（回測模組）
- 呼叫 Claude 或 GPT-4o API 進行 AI 分析
- 內建快取機制，避免重複呼叫外部 API
