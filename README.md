# 智投 AI — 全方位 AI 投資分析系統

一套整合即時市場資料、總體經濟數據、機器學習異常偵測與 AI 大型語言模型的投資輔助工具。透過網頁介面，不需要寫程式，就能使用 7 位 AI 分析師、多空辯論、DCF 估值、PE Band 標準差分析、孤立森林黑天鵝偵測、凱利公式倉位管理等進階投資分析功能。

---

## 系統需求

- Node.js 18 版以上
- 至少一個 AI API Key：Anthropic（Claude）或 OpenAI（GPT-4o）

---

## 本地安裝與啟動

第一步，安裝相依套件：

```
npm install
```

第二步，在專案根目錄建立 `.env` 檔案：

```
ANTHROPIC_API_KEY=你的 Claude API Key
OPENAI_API_KEY=你的 OpenAI API Key
FRED_API_KEY=你的 FRED API Key
```

三個 Key 中，AI Key 至少填一個。FRED_API_KEY 選填，沒有的話宏觀經濟模組僅能顯示 VIX，其餘總經指標無法取得。

第三步，啟動伺服器：

```
npm start
```

開啟瀏覽器前往 `http://localhost:3000`。

1. 將專案上傳到 GitHub Repository
2. 前往 render.com，點擊 New，選擇 Web Service
3. 連接 GitHub Repository，點擊 Connect
4. 填入設定：
   - Build Command：`npm install`
   - Start Command：`npm start`
   - Instance Type：Free
5. 新增環境變數：

   | 變數名稱 | 說明 | 必填 |
   |---|---|---|
   | ANTHROPIC_API_KEY | Claude API Key | 二擇一必填 |
   | OPENAI_API_KEY | OpenAI API Key | 二擇一必填 |

左側欄提供六個快速詢問按鈕：今日大盤分析、熱門投資題材、投資組合規劃、基本面指標、技術分析入門、存股策略。

自選股區域顯示五檔股票的即時報價，點擊直接發問。

右側面板提供快速分析工具，支援個股分析、新聞解讀、組合評估、大盤展望四種模式，即時查詢股價後送 AI 分析。

### 四、7 位 AI 分析師系統

這是整個系統最核心的功能。輸入股票代號，系統自動從 Yahoo Finance 抓取即時股價、財務數據、新聞 RSS，然後依序啟動 7 位各有專屬人格設定的 AI 分析師，以 Server-Sent Events 串流方式即時輸出報告，每位分析師完成就立即顯示，不需等待全部完成。

**六位專業分析師（並行）**

基本面分析師：財務三率（毛利率、營益率、淨利率）趨勢、負債比、自由現金流健康度、ROE 趨勢，輸出財務評分 1 至 10 分。

技術分析師：均線多空排列（月線、週線、日線）、量價關係、KD/RSI/MACD 訊號、K 線型態辨識（頭肩頂、旗形、杯柄等），給出進場點、停損點、目標價。

新聞情緒分析師：從 Yahoo Finance RSS 抓取最新新聞，對每則新聞進行情緒評分（-5 到 +5），判斷利多出盡或利空反彈的逆向訊號。

市場情緒分析師：結合 VIX 與技術指標判斷市場情緒極值，分析法人籌碼動向，評估是否出現反向操作訊號。

長線投資策略師：以巴菲特、費雪框架評估企業護城河（品牌、技術、網路效應、成本優勢），判斷產業未來 5 至 10 年趨勢，給出是否值得長期持有的明確結論。

短線交易員：專注 3 至 20 個交易日波段，找出催化劑（財報、法說、訂單、政策），提供完整交易計劃：做多或做空、進場區間、停損點、第一目標價、第二目標價、預期持倉天數。

**最終決策官（綜合）**

讀取上述六位分析師的完整報告，找出共識點與分歧點，進行裁決，輸出最終投資評等（強力買入/買入/持有/減持/強力賣出）、信心指數（1 至 10 分）、12 個月目標價區間、最大風險因素、觸發升評或降評的條件。

**分析時間：** 完整 7 位報告約需 30 至 60 秒，加辯論模式約再需 20 至 30 秒。

### 五、多智能體多空辯論

在 7 位分析師模式中選擇「多空辯論」，系統啟動三個獨立角色進行兩輪辯論：

第一輪：多頭分析師提出 3 至 5 個核心買入理由，空頭分析師逐一反駁。

第二輪：多頭針對空頭論點反擊，空頭給出最終總結，強調最關鍵風險點。

裁判裁決：中立裁判評估雙方論點的說服力與數據支撐，指出最關鍵的爭議點，給出中立投資建議，並標示目前偏多或偏空。

這個機制的目的是過濾 AI 的過度樂觀幻覺，讓分析更接近真實的投資委員會決策流程。

### 六、新聞情緒分析

輸入股票代號，系統從 Yahoo Finance RSS 抓取最新新聞標題，AI 對每則新聞進行情緒評分，計算整體情緒均分，並在情緒儀表板上以視覺化方式呈現（-5 極度悲觀至 +5 極度樂觀）。

分析內容包含：各新聞情緒評分、整體市場情緒偏向判斷、對股價短期衝擊預估、是否有利多出盡或利空反彈的逆向訊號。

### 七、AI 自動化估值模組（DCF 2.0 + PE Band 標準差）

輸入股票代號與財務數據，系統自動從 Yahoo Finance 取得即時股價、EPS、自由現金流，並從五年月線歷史數據計算 PE Band 標準差區間，再由 AI 執行完整估值。

**PE Band 標準差分析**

取近五年每月股價除以當前 EPS，計算歷史本益比的均值（mean）與標準差（σ）。依此計算五條價格帶：-2σ（超低估）、-1σ（低估）、均值（合理）、+1σ（偏高）、+2σ（超高估）。顯示目前股價位於五年歷史的幾個百分位，以及落在哪個標準差區間。

這比單純的歷史最高最低更有統計意義，因為極端值不會扭曲合理區間的判斷。

**DCF 2.0 現金流折現模型**

自動讀取公司近期自由現金流（FCF），動態估算 WACC（加權平均資本成本，依負債比自動計算）。預測未來五年現金流時採用逐年遞減 15% 的成長率，而非固定成長率，更貼近現實。終值採用永續成長率（最高 3%），折現後計算公司總估算價值與每股內在價值。

AI 綜合 PE Band 位置與 DCF 每股內在價值，給出建議買入區間（強力買入價、合理買入價、過熱警戒價）。

### 八、技術指標分析模組

### 九、回測分析模組（含 Sharpe Ratio、Max Drawdown、Kelly Criterion）

描述交易策略的買入和賣出條件，系統先從 Yahoo Finance 抓取回測期間的真實歷史數據，計算以下量化指標作為基準：

**真實計算的量化指標**

夏普比率（Sharpe Ratio）：超額報酬除以波動率，衡量每承擔一單位風險的報酬。小於 0 不如無風險利率，0 至 1 尚可，1 至 2 良好，大於 2 優秀。

最大回撤（Max Drawdown）：持有期間從高點到低點的最大跌幅，代表最壞情況下的虧損幅度。

卡瑪比率（Calmar Ratio）：年化報酬除以最大回撤，數值越高代表每承擔一單位回撤所獲得的報酬越高。

凱利公式建議倉位：根據歷史勝率（上漲日佔比）與盈虧比（平均漲幅/平均跌幅）計算 Full Kelly 與 Half Kelly 建議的資金比例。

**AI 回測失敗案例分析**

除了績效數字，AI 會找出三種市場環境下策略會徹底失效的情境，說明失敗根本原因、發生頻率、以及如何改進策略應對。

### 十、凱利公式倉位管理

獨立的倉位計算模組。輸入策略的勝率、平均獲利百分比、平均虧損百分比、總資本、最大倉位上限，系統計算：

Full Kelly：理論上最大化資本成長的最佳倉位比例。

Half Kelly（建議使用）：Full Kelly 的一半，大幅降低資金曲線波動，在長期操作中更為穩健。

四分之一凱利：保守型投資人適用。

每筆建議倉位金額、期望值（每筆交易的數學期望獲利）、賠率比。

AI 解讀包含：這個勝率與盈虧比的策略品質評估、為何使用 Half Kelly、連虧情境下的資金保護策略、不同風格投資人的倉位調整建議、使用凱利公式的三個常見陷阱。

**凱利公式的限制**
Kelly Criterion 假設每次交易結果獨立，且勝率與盈虧比穩定不變。實務上這些假設往往不成立，因此 Half Kelly 是更安全的選擇。倉位上限設定（預設 30%）也是重要的風險保護機制。

### 十五、黑天鵝預警系統（孤立森林 + 滾動 Z-Score）

使用兩種演算法同時偵測 VIX 與個股的異常波動，計算風險等級並觸發警報。

**孤立森林（Isolation Forest）**

機器學習無監督異常偵測演算法，以 JavaScript 在後端實作（100 棵樹，256 個樣本）。原理是：正常數據需要較多次分割才能被孤立，異常數據很快就能被孤立。分數接近 1 表示高度異常，接近 0 表示正常。

本系統使用「日報酬率」與「成交量標準差」兩個特徵組成二維特徵向量，輸入孤立森林進行多維度異常偵測。

孤立森林分數說明：
- 小於 0.5：正常
- 0.5 至 0.65：輕微異常
- 0.65 至 0.8：異常，需留意
- 大於 0.8：極端異常，建議啟動風險管理

**滾動 Z-Score**

計算過去 252 個交易日的滾動 20 日均值與標準差。當某日數值偏離 20 日均值超過 2.5 個標準差（|Z| ≥ 2.5）時視為統計異常。同時偵測價格日報酬率與成交量兩個維度。

兩種演算法同時觸發時，黑天鵝風險大幅提升。

**AI 分析內容**

整體風險評級（低、中、高、極高）與判斷邏輯、孤立森林分數與 Z 值的綜合解讀、歷史類比（此信號水準後市場常見走勢）、具體操作建議（是否降低倉位、增加避險部位）、Emergency Stop 觸發條件（什麼情況下應立即全數出場）。

## 常見問題

**按下分析按鈕後沒有反應，或顯示「伺服器錯誤」**

最常見原因是 Render 環境變數未正確填入 API Key。至 Render Dashboard 的 Environment 頁面確認 ANTHROPIC_API_KEY 或 OPENAI_API_KEY 已填入，且沒有多餘空格。

**分析結果顯示「分析失敗：Anthropic API key 未設定」**

同上，確認環境變數設定正確。本系統按鈕有倒數計時顯示，若按下後秒數持續增加代表正在處理，不是當機。

**7 位分析師需要多長時間？**

完整七位分析師報告約 30 至 60 秒。加上多空辯論約再需 20 至 30 秒。每位分析師完成就會立即顯示，不需等待全部完成。

**黑天鵝偵測需要 10 至 15 秒**

正常現象，系統需從 Yahoo Finance 抓取三個標的各一年的每日歷史數據（約 750 至 756 筆），再執行孤立森林（100 棵樹）與 Z-Score 計算。

**宏觀經濟模組顯示「FRED API 未設定」**
